Launch a critical event

You can create a critical event using an empty form or a template your organization has configured for a particular situation. You can launch a critical event by clicking Launch a New Event on any section under the Events tab.

If you use a template, most configuration is unnecessary unless you need to change something—fields are prefilled, task lists are included, communication templates are preconfigured, and reference materials are uploaded. If there are required fields that are missing information, you're informed when you try to launch the event. For details on critical event templates and how to create them, see Creating a Critical Event Template.

Exercise mode

The Exercise Mode toggle lets you want to launch the critical event in Exercise Mode. When you use this mode, the critical event is sent to the selected contacts to allow recipients to practice receiving and responding to messages similar to those they'd see in a real emergency. When you launch an event in Exercise Mode, any communications sent have the [DRILL] tagged added to the subject and body so recipients know it is an exercise.

Step 1: Select a template—or not

To use a preconfigued template, select it under Templates. Otherwise, you can continue without selecting one to create an ad hoc event.

Step 2: Set the title and event type

Complete the critical events sections to define the event properties, associated task lists, the communications you want to sent message you want to send, and what documents are linked to the event.

Edit the title and description

The title and description help you understand the impact of the event at a glance and uniquely identify this event in the system. These fields may be prefilled if you're using a template. However, you can edit them to be more descriptive.

Select the event type

Select the Event Type that best matches the critical event you want to launch.

Step 3: Set event details

  1. Select the Owner of the event. **purpose**
    1. To assign an owner, ***
    2. To replace an existing owner, click Edit and select a different name, or remove the current owner’s name by clicking Remove.
  2. Set the location, if relevant for the event.
    1. Location: Change the location by clicking Edit an address, then select a new location from the Select a Location dialog. Or, click Use Shapes on the Map. When you click Select this Location, the latitude/longitude are copied to the clipboard.
    2. Location Name — Enter a location description, such as an address.
  3. Set the time Zone, start time, and (if needed) end time.
    1. Local Event Time Zone — Select the time zone from the drop-down list.
    2. Local Event Date/Time — Click the Calendar icon to select the date and time.

Step 4: Configure custom fields

  1. Click Add Custom Fields to add incident variables to provide additional information to the response team.
    • Search and select the checkbox next to the desired Incident Variable names.
      • Geo Point, Location: Geo Shapes, and On-shift Date Range are not supported.
    • If the variable is required, select the Req checkbox.
    • Click Add.
    • To reorder the custom field, click and hold the menu icon beside it and drag it to its new position.
    • To remove a custom field, click X to delete it.

Step 8: Launch the critical event

After you confirm that the critical event settings are what's required for your scenario, you're ready to launch the event.

  1. Select the Turn off notification for task assignment and dashboard sharing checkbox to turn off the Notification when launching the event.
  2. Select the Turn off task list Notifications checkbox if you want to turn off task list Notifications. The default is on. **where is this**
  3. Click Launch to launch a new critical event.
