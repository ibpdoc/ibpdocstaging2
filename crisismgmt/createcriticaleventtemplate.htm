Critical Event Templates

Critical event templates let you pre-configured the components of a critical event for specific scenarios that impact your organization and contacts, saving time and ensuring consistency and completeness during high pressure launch situations.

Create or edit a critical event template

When creating a critical event template, you set up the default values for the launch form that operators fill out at launch time, as well as associate task lists, communications, and documents to the template. Depending on the permissions you give, the operator can edit these if needed when launching the critical event.

  1. Select Critical Event Templates from the Critical Events tab.
    1. To create a new template, click New Critical Event Template.
    2. To edit a template, click the Pencil icon beside the template name.
  2. Set the status of the template. New templates start in Draft status so they don't accidentally get used before their ready.
  3. Fill in the Critical Event Template details accordingly. Required fields have a red asterisk (*). You can fill in the fields now and return later to edit the template when you have additional information.
  4. Add existing task lists, communications, or documents with the template,
  5. Click Save when done—if the template is ready to use, make sure to set the status to Live before saving.

Add and configure custom fields

You can add and configure custom fields using any incident variables. Preconfiguring these fields helps reducing the time to launch during a critical event.

  1. Click Add Custom Fields.
  2. Select the variables you want to add. You can only use on method at a time to add variables, but you can return to the Add custom fields dialog box to add more.
    • By Incident Variables: Select individual variables to add then click Add.
    • By Incident Template: Add all variables associated with the selected incident template. This is useful if you want to duplicate much of the functionality of another template.
    • By Existing Incidents: Add all variables associated with an existing incident (this can help you turn an ad hoc event into a template).
  3. After you add the custom fields, configure their default values.
  4. Finish filling out the template and click Save.

Add Custom Fields

Your template can include your Organization's incident variables so that the Critical Event is automatically filled in when launched.

To add custom fields to the template:

  1. Click Add Custom Fields.
  2. Search and select the checkbox next to the desired incident variables name(s).
  3. If required, select that checkbox as well.
  4. Click Add. The custom field is added to the bottom of the template.

To reorder the custom field, click the Hamburger icon and while holding the mouse, drag the field to its new position.

If the custom field was created by mistake, click X to delete it.
Folder When you launch a Critical Event, a folder name can be included in the template. The launched event is placed in the folder. A default folder is presented, but you can change it. If the user has no folder permission, then the event will be placed in the Uncategorized folder.
Turn off Notifications for task assignment and dashboard sharing Select this checkbox to turn off the Notification when launching the event and dashboard sharing.
Auto-Launch Select the checkbox to auto-launch the selected task lists and incident communications.
Task Lists Click Add to add existing task lists. Select the desired checkbox and click Save. If already added, you can also delete a task list by clicking the X next to its name.
Incident Communications Click Add to add existing Incident Templates. Select the desired checkbox and click Save. If already added, you can also delete an incident template by clicking the X next to its name.
Documents Click Add to add existing Documents. Select the desired checkbox and click Save. If already added, you can also delete a document by clicking the X next to its name.

View and manage critical event templates

You can see the existing critical event templates by going to the Critical Event Templates subtab and clicking on the name of the template you're interested in.

  • You can also move the individual files into folders or add subfolders.
  • To delete a Critical Event template, select the checkbox next to its title, then click Delete.
