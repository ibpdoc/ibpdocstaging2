Release 6.4.2.0 Private Cloud

Infinite Blue®Platform Version 6.4.2.0 Release Notes

Release Date

September 2022

Fixed Issues

Following is the list of fixed issues with this release:

Support ID

Bug ID

Public Description
PTSP-3800 IB-17425 #link tokens for templates was throwing cannot cast into DataObject error.
PTSP-3724 IB-17229 Button shows label only in one language across all Users (who have different languages).
PTSP-2302 IB-13085 Date picker for filters was picking different localization settings from main UI pages and admin pages, leading to error while parsing date time value set from admin page on main UI page.
PTSP-3661, PTSP-3793, PTSP-3422, PTSP-3806 IB-17475 Default License Key platform ships with for fusion charts was expired.
PTSP-1174 IB-10529 Field/triggers have facility to add validation rule and also optionally treat them as warning; and ignore during runtime. The warning facility was not implemented during records restore process and there was no way to optionally skip such validation rules (promoted as warning)
PTSP-3728 IB-17216 In-Use Connections exceeding max life time or max time in use would go in both 'in use' and 'has error' states and wont be cleaned up organically from connection pool.
PTSP-3398 IB-16073 Inline edit of text area field showed encoded characters as &#38; or &#35; instead of &, # for few HTML characters.
- IB-17293 Platform was creating connection pool over all databases even in single machine/dedicated PROD nodes. These connection pools would remain unused on multi-node setup with single node running for dedicated PROD component, otherwise the connection pool would have been reused by other components running on same node.
PTSP-2033 IB-16734 Single record delete from list view was not displaying 'record name' in pop up confirmation message. Workaround was to pull record name field on list view. Fixed this to always display record name in pop up message.

Enhancements

Following is the enhancement with this release:

  • Microsoft Exchange - OAuth Integration: Platform now supports Modern Authentication using OAuth 2.0 for Exchange emails services. For more information, see Enabling Microsoft Exchange Integration.

    Note:

    • As announced here, basic authentication in Exchange Online will be soon deprecated. Platform customers are hereby recommended to start using Microsoft Exchange OAuth.

    • This feature cannot be used in conjunction with Support Admin Login or Login As user sessions.

Known Issues

Following is the list of known issues with this release:

  • [OAuth 2.0 Exchange] [Intermittent]: Emails fail to send with Connection reset error.

    Workaround: Try resending the email.

  • [Email]: If the AdminEmail and the attached email for Exchange OAuth server are different, the following error is thrown and fails to send any Welcome email.

    The user account which was used to submit this request does not have the right to send mail on behalf of the specified sending account., Cannot submit message.

    Workaround: Ensure both the AdminEmail and the attached email are same.
