Known Issues

Following is the list of known issues with this release:

  1. Multi-Factor Authentication – When Authentication Mode is set to 'Global' and Enabled Single User Multi-Tenant facing issue in login and Switch tenant flows.

    Workaround: In case of Global authentication all tenants must have MFA (multifactor authentication) enabled. So, user should be not able to switch between 2 different authentication patterns.

  2. Export to XLS – Invalid row number (65536) outside allowable range (0..65535)

    Workaround: This issue will only reproduce if exporting more than 65535 rows or 255 columns in XLS format. As interim workaround: please change output format to xlsx. This is a limitation from apache-poi and pre-2007 excel formats which can be mitigated by forcibly exporting to post 2007 excel formats

  3. Charts on Postgre DB – When the "GroupBy" is "Day" then the dates are not shown correctly in the chart.

    Workaround: No Workaround currently.

  4. Record Name Template – If Expression field is used in a Record Name Template record name is not evaluated correctly.

    Workaround: No Workaround currently.

  5. Marketplace app Update – Push Updates button is disabled intermittently.

    Workaround: Reload the master cache.

  6. REST Trigger – Generated code in rest service trigger is not updated without opening mappings tab.

    Workaround: Every time you update the trigger open the mappings window and save.

  7. Document Template – Cloning a document template report doesnt carry the template.

    Workaround: Upload the template manually after cloning.

  8. BatchJob – Deleting object type is not clearing up the batchjob.

    Workaround: When user deletes a Object type, he should go and delete corresponding batch job as well.

  9. Versioning with GIT(SDLC) – Install/Update from GIT lists empty directories.

    Workaround: It just a UI issue no impact on functionality.

  10. Versioning with GIT(SDLC) – New branch at tenant level can’t be used, unless using BRANCH option at instance preferences page.

    Workaround: Configure the GIT repository properties at Instance/Master preference level.
