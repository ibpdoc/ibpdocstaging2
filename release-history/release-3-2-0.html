Release 3.2.0

Infinite Blue® Platform Version 3.2 Release Notes

The following sections describe known issues and issues fixed in this release.

Known Issues

PSC00327364 Missing link when creating an ISV customer with this same email as the ISV partner
When creating an ISV customer whose email is the same as the ISV partner, the link to switch to the ISV customer does not appear in the drop down menu. To accomplish this, user ISV partner must log out and log back in.

PSC00327507 "Choose Tenant" link is not working in the legacy header
Non-administrator users who are part of multiple tenants whose applications use the legacy header and footer will not be able to switch to another tenant in the legacy header. The user must navigate to the Pacific header application to change tenants.

PSC00328535 Some areas in Platform contain incomplete translations
legacyObjects.xml and listItems.xml contain some objects whose names are not translated Some messages that are displayed when exceptions are thrown by Platform are not translated

PSC00328987 Edit/in-line edit for Korean and Chinese languages do not work properly
The edit/in-line edit of Date-Time fields using a date-picker does not work properly in the Korean and Chinese language translation of Platform. To edit/in-line edit a Date-Time field, manually change the day/month/year/hour/min values.

PSC00296220 When using Firefox, an error message is displayed when the mobile emulator runs and FORM based authentication is configured.
When FORM based authentication is configured for a session-enabled application and run in the mobile emulator using Firefox, the following error message is displayed:

Error retrieving catalog 'https://xxxxcatalog.json'. Http status: 500. Status (from jqXHT) : 500 statusText (from jqXHT) : Internal Server error
