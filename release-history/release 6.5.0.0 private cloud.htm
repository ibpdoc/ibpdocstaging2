Known Issues

Following is the list of known issues with this release:

  1. [Installer] [Java] - Platform 6.5 installer by default is bundled with Java 11 yet the latest supported Java runtime version is 17.

    Workaround: The installer setup can be manually changed to Java 17.

  2. [Picklist] [App XML] - In application XML, the shared pick-list shows error information for one object only.

    Workaround: As one object is showing the error information, it is possible to fetch error information for other objects as it is shared.

  3. [Picklist] [App Publish] - Applications can be Published or Publish updates even if the pick-list contain duplicate codes or values.

  4. [Picklist] [Country Code] - Duplicate country code prompts an error.

    Workaround:

    • Country picklist field in LDF/User/Customer objects may have duplicate code "MO" for countries - Micronesia and Macao. Please fix this manually by updating code for Micronesia to "FM".

    • For fixing this on any new tenants, Master Admin should log in to Master Zone > System Console > Control Panel > List Items, and manually change code against "Micronesia" to "FM" and Save.

  5. [Security Levels] [Password Strength] - Few passwords display different strengths on front-end vs back-end.

  6. [Security Levels] [Password Strength] - Password Strength UI validation is not working on Portal change password flow.

  7. [Logs] - Incorrect Max idle time printed in cleanup log for some connections.
