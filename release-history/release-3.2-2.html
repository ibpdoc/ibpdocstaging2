Release 3.2.5

Infinite Blue® Platform Version 3.2.5 Release Notes

The following sections describe known issues and issues fixed in this release.

Fixed Issues

17433 No relationship definition found message in chart view

No relationship definition found message in chart view after installing XML file for the application while the original application has the correct relationship

17666 Add new clone ServerSide API

Added a new ServerSide API to clone a given record.  Added this to template helper dropdown.

17704 Importing an app using an XML containing more than 15K records

When importing an app using an XML containing more than 15K records as seed records using an OE database throws an error message

16606 No relationship definition found message in chart view

No relationship definition found message in chart view after installing XML file for the application while the original application has the correct relationship 

12619 Seed Record Enhancements

  • Performance Improvement in Seed Record Handling in terms of page load times
    • Moved from being a section in App details page to dedicated page
    • Implementation pagination
    • Moved to AJAX loading from full page reload on removal of records
  • Ability to attach seed record from all pages in record selector
  • Ability to filter seed records based on object type
  • Fixed seed record list and its count going out of sync
  • Fixed issue with removal of large number of records at a time
  • Max Seed records is now controlled by shared.property (MaxSeedRecords). Default is 15000
  • Fixed issue where Seed records were not getting deleted when the original record is deleted

12624 OESO Enhancements

The OpenEdge Service Object has been enhanced to support both server side select query and client side select query.

00321834/PSC00339275 Workflow Status Field blanked out when importing spreadsheets

The workflow status field is blanked out on imports where the “Treat empty spreadsheet cells as NULL values” is checked.

00316849/PSC00336962 Bad performance login command in SOAP and REST

We have not been able to reproduce this issue. Should you experience a performance issue, please set EnableLoginDiagnostics=true, and run your application. When the problem occurs, attach the log file loginDiagnostic.log to the support case.

00320616/PSC00340146 PD4ML cannot support the THAI language

THAI characters in PDF templates are not displayed correctly.

00308989/00316849/PSC00332223 Expression x formula in viewers

When using lookup fields in a viewer with Expression as filter conditions, Platform creates a weird query

00312944/00313824/PSC00333179 Unique auto number fields get reverted

Unique auto number fields that are set to not allow duplicate values are reverted back to a value that was lower than the value already assigned to existing records.

00313824/PSC00333504 Auto number inconsistent

Auto number field is working incorrectly; sometimes jumping by 1000

00319429/PSC00336887 Auto-number starting number does not work

Auto-number "starting number" property does not work.  After changing the "Starting Number", the change does not take effect.

17596 Auto-number is jumping by 3

Auto-number is jumping by 3 when marked as read-only.
