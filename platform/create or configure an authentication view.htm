Create or configure an Authentication view

To access Platform data from a mobile app, users must log in to Platform using their Progress ID and password. The Authentication view provides this functionality.

If you already created a Progress Data Service provider and selected Add an "Authentication" view to my app, select the view and start with Step Review and change the following General Settings:Optionally, change the name of the view by editing the Name field. The app code will refer to this name.Note: You can only modify this field before clicking Apply.Optionally, change the value in the Title field, for example to Log In, which changes the label for the view in the left pane and specifies the heading that displays in the app screen for this view.Expand the Navigation node and select Include this view in the app navigation. By default, an icon is selected to represent this view in app navigation. Optionally, you can change this icon by opening and selecting a different icon, as shown below:Expand Data Binding and verify that a Progress Data Service provider is selected, or create one if you have not already done so. See Add a Progress Data Service provider for details.. Otherwise, to create an Authentication view for a Progress Data Service provider, start with Step With the app open in the Views window, select the Manage Views tab and click Add View::

  1. With the app open in the Views window, select the Manage Views tab and click Add View:

  2. From the palette of view types, select Authentication:

    The new Authentication View properties display in the center pane, with the General option selected to show the General Settings:

  3. Review and change the following General Settings:
    1. Optionally, change the name of the view by editing the Name field. The app code will refer to this name.
      Note: You can only modify this field before clicking Apply.
    2. Optionally, change the value in the Title field, for example to Log In, which changes the label for the view in the left pane and specifies the heading that displays in the app screen for this view.
    3. Expand the Navigation node and select Include this view in the app navigation. By default, an icon is selected to represent this view in app navigation. Optionally, you can change this icon by opening and selecting a different icon, as shown below:

    4. Expand Data Binding and verify that a Progress Data Service provider is selected, or create one if you have not already done so. See Add a Progress Data Service provider for details.
  4. Under Authentication View, click the Sign In option to scroll to the Sign In Screen properties, then expand and change the settings as follows:
    1. Expand Email. In the Textbox label field, optionally change the label you want to appear for the field representing the user name. For example, you might enter Progress ID for connecting to a Progress Data Service for Platform. Optionally, enter text in the Help text field. This text will appear in the field in the app.
      When editing a view, you can click Apply at any time and test it in the device simulator. For example, if you click Apply now, the screen below shows the Authentication view after typing Progress ID (along with the previously suggested settings):

      Note: The Authentication View properties scroll back to the General Settings after you click Apply. Note also that the Register button, if it appears as displayed in the simulation, is not needed for this example app.
      Reviewers: Is it possible for users to register for a Progress ID and access to Platform from a "Register" screen or should this always be disabled?
    2. To remove the Register button, expand Configuration / Options and deselect Enable "Register" screen. This disables the Register button from the view—click the Sign In option, again, to continue.
      Reviewers: Is it possible for users to register for a Progress ID and access to Platform from a "Register" screen or should this always be disabled?
    3. Expand Password. In the Textbox label field, optionally change the label you want to appear for the field representing the password. Optionally, enter text in the Help text field. This text will appear in the field in the app.
    4. Expand Confirm button. Optionally, change the label you want to appear on the confirm button, for example, to Log In.
    5. Expand Redirect and select the view to display after a successful log in, if you have created one. (We return to this step, later, after creating a Master Detail view.)
  5. Click Apply.
    The device simulator displays your changes similar to the following screen, which shows the completed Authentication view in the device simulator with the device iPhone 6 selected:

    To test the view, enter a valid Progress ID and password in the device simulator.
